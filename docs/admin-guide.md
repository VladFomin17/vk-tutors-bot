# Руководство администратора

## Подключение сообщества и новой VK-беседы

Один раз для сообщества VK:

1. Включите сообщения сообщества.
2. В разделе API включите Bots Long Poll API и событие `message_new`.
3. Создайте токен сообщества с правом управления сообщениями и запишите ID сообщества и токен в `VK_GROUP_ID`/`VK_ACCESS_TOKEN`.
4. Перезапустите `vk-listener` и убедитесь по логам, что появилось `VK Long Poll connected`.

Для каждой новой учебной беседы:

1. Добавьте сообщество в беседу и разрешите ему доступ к переписке.
2. Отправьте в беседу новое обычное сообщение. Listener обнаружит беседу и синхронизирует её участников.
3. В панели откройте **Учебные группы**, создайте группу и в блоке обнаруженных бесед привяжите нужную беседу.
4. Откройте группу и назначьте участникам роли: **Первокурсник**, **Тьютор** или **Староста**. Неопределённые участники не попадают в рассылку.
5. Проверьте число первокурсников и выполните тестовую рассылку.

Одна беседа связывается только с одной учебной группой. Новые сообщения обновляют состав; вышедшие участники становятся неактивными, но история рассылок сохраняется.

## Ежедневная работа

### Группы и студенты

На странице групп видны связанная беседа, количество студентов, неопределённых участников и последняя активность. Глобальный список студентов поддерживает поиск и фильтр по группе/активности. Перед первой рассылкой классифицируйте всех нужных получателей.

### Создание рассылки

1. Откройте **Рассылки → Создать**.
2. Укажите название, текст, при необходимости ссылку, дедлайн и тип подтверждения.
3. Выберите одну или несколько групп с подключёнными беседами.
4. Проверьте предпросмотр и создайте рассылку.

Состав получателей фиксируется в момент создания. Позднейшее изменение ролей не меняет уже созданную рассылку. Основное сообщение отправляется сразу, напоминание — за 24 часа до дедлайна, если этот момент ещё не прошёл.

### Контроль доставки и ответов

На странице рассылки показаны основная отправка и напоминание по каждой беседе: состояние, число попыток, время и последняя ошибка. После устранения причины окончательно неудачное задание можно вернуть в очередь до дедлайна.

Ответ принимается согласно выбранному типу подтверждения. Поздний ответ сохраняется с отдельной отметкой. Изображения доступны только авторизованному администратору. Результаты выгружаются в XLSX или DOCX.

### Удаление

Удаление рассылки безвозвратно удаляет её локальные задания, результаты и изображения. Уже отправленные сообщения VK остаются. Перед удалением при необходимости скачайте экспорт и убедитесь в наличии backup.

## Безопасная работа

- Используйте отдельную учётную запись администратора и не передавайте пароль/токен VK в чатах.
- Завершайте работу кнопкой выхода, особенно на общем компьютере.
- Не открывайте PostgreSQL и Docker API в интернет.
- При подозрении на утечку немедленно сообщите оператору сервера для ротации секретов.
- Операции сервера, backup/restore и диагностика описаны в [production-инструкции](production.md).
