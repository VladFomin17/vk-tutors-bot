# VK API Feasibility Spike

Дата проверки: 14 июля 2026 года.

## Проверенная конфигурация

- VK API `5.199`.
- Токен сообщества.
- Bots Long Poll API.
- События `message_new` и `message_reply`.
- Многопользовательская беседа, в которую добавлено сообщество.

## Результаты

1. `groups.getLongPollServer` работает с токеном сообщества.
2. Новые сообщения беседы приходят как `message_new`.
3. Беседа может отсутствовать в `messages.getConversations`, пока бот уже получает
   её события. Поэтому Long Poll является основным механизмом обнаружения бесед.
4. `messages.getConversationMembers` доступен по `peer_id` беседы и возвращает
   пользователей и сообщества.
5. `messages.send` успешно публикует сообщение с `@all`; массовое упоминание
   подтверждено в клиенте VK. Метод возвращает `0` вместо пригодного
   идентификатора сообщения.
6. Входящие сообщения беседы также имеют `message_id = 0`, но содержат уникальный
   в пределах беседы `conversation_message_id`.
7. Reply-событие содержит `reply_message.conversation_message_id` и автора
   исходного сообщения.
8. Переданный в `messages.send` JSON `payload` полностью возвращается внутри
   `reply_message.payload`. Уникальный `broadcast_token` в payload однозначно
   связывает ответ с исходной рассылкой или напоминанием без видимого кода в тексте.
9. Текстовый reply и reply с вложением `photo` получены корректно.
10. Фотография доступна для скачивания; MIME, размер и сигнатура JPEG подтверждены.
11. `messages.getHistory` с токеном сообщества для тестовой беседы возвращает
    `Access denied`, поэтому он не используется для восстановления ID отправки.

## Архитектурные решения

- Беседы обнаруживаются из Long Poll и сохраняются как `vk_chats`.
- Учебная группа связывается с ранее обнаруженной `vk_chat`.
- Каждый `outbound_message` получает криптографически случайный
  `broadcast_token`, передаваемый в VK message payload.
- Ответ принимается только когда:
  - он является reply;
  - исходное сообщение отправлено текущим сообществом;
  - payload содержит существующий `broadcast_token`;
  - автор ответа присутствует в снимке получателей.
- Идентификатор входящего сообщения: `(peer_id, conversation_message_id)`.
- `vk_message_id` сохраняется как полученное значение, включая `0`; бизнес-логика
  на него не опирается.
